# Product Docs (canonical)

This folder contains the product-facing specification artifacts for this repo.

## How to use

- Product owns the content here.
- Engineering treats these docs as canonical inputs.
- Engineering may add clarifying notes/questions, but should avoid rewriting intent.

## Files

- 00-vision.md – north star: problem, user, success
- 01-problem-and-users.md – detailed user segments + pain
- 02-use-cases.md – prioritized use cases + edge cases
- 03-flows/ – per-feature flows (happy/error/empty states)
- 04-rules-and-policy.md – business rules, precedence, auditability
- 05-acceptance-criteria.md – testable requirements
- 06-nonfunctional.md – security, perf, reliability, compliance
- 07-open-questions.md – tracked gaps
- 08-analytics-and-events.md – event taxonomy + KPIs
- 09-glossary.md – shared definitions
