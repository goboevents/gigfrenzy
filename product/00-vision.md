# Vision

## One-liner

GigFrenzy (GF) lets consumers **book their dream event. today.** Start by choosing your event type, ZIP, and date/time—then pick your primary experience (DJ, Karaoke, or Photo Booth). Build a custom package in a tactile card-based builder, see an instant estimated quote, and book by submitting a request a vendor can confirm in their portal—without dozens of inquiries and back-and-forth.

## Problem

- Today: Consumers planning events search the internet/social, contact vendors, and manage fragmented replies, pricing, and availability across email/DMs.
- Impact: Hours of wasted time, delayed decisions, and higher risk of mismatch/no-shows—making event planning stressful.

## Target user(s)

- Primary: Consumers planning a live event (birthday party, wedding, banquet, etc.) who want a fast, confident way to book an event experience.
- Secondary (later): Event planners/agencies coordinating vendors across multiple events.

## Why now

Booking expectations have shifted to instant, confidence-building experiences. Discovery is online, but booking remains manual and fragmented—GF can unify package selection, estimate pricing, and booking confirmation in one flow.

## What success looks like

- Metric(s):
  - Time-to-package (minutes)
  - Time-to-book-request (minutes)
  - Vendor confirmation time (hours)
  - Package→booking conversion rate
  - Quote estimate engagement (builder completion rate)
- Qualitative:
  - “This felt fun and easy—like building my event experience.”
  - “I knew what I was getting and what it would cost (roughly).”
  - “I felt confident the vendor could actually deliver what I picked.”

## Scope

### v1 (must-have)

- Consumer-first experience; planners later
- Experience-first package builder:
  - user selects event category/type
  - user enters ZIP code (location anchor; ZIP-only in v1)
  - user selects event date/time
  - user selects a **primary service** (locked): DJ, Karaoke, Photo Booth
  - user selects tier (low/standard/high; global)
  - user customizes via cards (styles/services/add-ons)
  - system produces an **estimated quote** + breakdown
- Matching: show eligible vendors who can fulfill required selections (service area/capabilities)
- Booking request + vendor confirmation in vendor portal (no contracts/payments in v1)
- Open vendor signup in v1; basic profiles + lightweight trust signals

### v2 (nice-to-have)

- Terms + payments (deposits/refunds) + contracts
- Stronger trust & safety (verification, quality checks)
- Real-time availability + instant booking (where possible)
- Planner/agencies workflows
- Smarter matching/recommendations (explainable)
- Full location capture (address/venue)

### Out of scope

- Full event management suite (invitations, seating charts, etc.)
